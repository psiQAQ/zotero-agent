# Web of Science Starter API 实施计划

> 当前执行目标以本计划为准。开发过程中在“进度”表更新事实、验证结果和必要调整；不把聊天记录写入文档。

## Goal

在不增加运行时依赖、不破坏现有通用导入能力的前提下，为 Zotero Agent 增加 plan 感知、受速率与本地额度保护的 `search_web_of_science` MCP 工具，并完成中英文文档、XPI 部署和真实 API 验证。

## Success Criteria

- `search_web_of_science` 只在启用时出现在 `tools/list`，返回规范化 JSON。
- 用户可在 Zotero 面板填写 API Key、选择实际 plan 并测试连接。
- 所有请求遵守 plan 间隔、单次安全上限和本地日计数；429 不自动重试。
- API Key 不出现在 URL、日志、MCP 输出或错误消息。
- `import_by_identifier` 保持通用；没有可复现缺口则零修改。
- 中英文指南回答个人申请、Key 获取、plan 区别、限制和本次更新；对应 README 有入口。
- 单测、构建、lint、Zotero selfTest 和真实最小查询均有明确结果。

## Constraints

- 固定 Clarivate HTTPS 域名和 `X-ApiKey`，不引入官方 SDK 或 XLSX 依赖。
- 不实现 Expanded API、专用导入工具、后台同步、缓存或隐式重试。
- 生产修改采用 TDD；一次完成一个可验证逻辑阶段并提交。
- 真实 API 验证前，Key 只能由用户写入 Zotero password 输入框，不能发到聊天或提交到仓库。

## 已确认的策略调整

官方没有公开可依赖的 plan/剩余额度自动识别接口，因此采用显式 `wos.plan` 下拉框：

| plan | 官方限制 | 插件请求间隔 | 插件单次安全上限 |
| --- | --- | ---: | ---: |
| Free Trial | 1/s，50/day，无 times-cited | 1100 ms | 50 records |
| Free Institutional Member | 5/s，5,000/day，有 times-cited | 220 ms | 500 records |
| Free Institutional Integration | 5/s，20,000/day，有 times-cited | 220 ms | 1,000 records |

插件另以 UTC 日期维护保守的本地请求数。该计数不是远端剩余额度，无法看见其他客户端对同一 Key 的使用。
